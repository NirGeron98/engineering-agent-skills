# TDD Workflow Checklist

- [ ] The smallest observable behavior and public interface are named.
- [ ] Existing test conventions, fixtures, helpers, and focused test command were inspected.
- [ ] The first test is minimal and would fail for the intended reason.
- [ ] Red result was observed or explicitly requested, the failure message/assertion mismatch was recorded, and the failure matches the intended behavior (not an unrelated setup, import, or type error).
- [ ] Implementation is the smallest step needed to pass the failing test.
- [ ] Refactor notes happen only after green.
- [ ] Focused and relevant broader verification commands are run or requested.
- [ ] Missing repo access, command output, or test runner evidence is marked as pending.
