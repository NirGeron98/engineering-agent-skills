# Metrics and Alerts Review Checklist

- [ ] Alert inventory is built from actual alert definitions, not memory or assumption.
- [ ] Signal/noise findings are backed by paging history, not intuition.
- [ ] Missing coverage is checked against known critical failure modes.
- [ ] Threshold recommendations cite historical data ranges.
- [ ] Ownership/escalation gaps are explicitly identified.
- [ ] Verification plan is included (test firing, review period).
- [ ] Missing alert configs, paging history, or metrics sources are marked pending.
- [ ] Noisy alerts are not accepted as normal without a finding.
