# Changelog Writer Checklist

- [ ] Audience, version/date, source range, and repository changelog format are identified.
- [ ] Diffs, commits, PRs, issues, tags, migration notes, docs, and tests were inspected or requested.
- [ ] Entries are grouped into features, fixes, breaking changes, migrations, deprecations, internal changes, and known issues as appropriate.
- [ ] Every entry traces to actual evidence.
- [ ] Breaking changes and upgrade actions are explicit.
- [ ] Internal-only changes are labeled or omitted from user-facing notes.
- [ ] Unreleased, speculative, or unverified work is excluded or marked pending.
- [ ] Missing diff, commit range, PR list, or release scope is marked pending.
