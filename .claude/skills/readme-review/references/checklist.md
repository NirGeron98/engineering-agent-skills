# README Review Checklist

- [ ] README audience, project purpose, and primary tasks are identified.
- [ ] Manifests, scripts, CI, env examples, docs, code entry points, examples, and tests were inspected or requested.
- [ ] Setup, install, run, test, environment, and troubleshooting claims are verified or marked pending.
- [ ] Missing sections are tied to user tasks, not generic completeness.
- [ ] Stale or unsupported claims cite the conflicting evidence.
- [ ] Suggested structure is concise and task-oriented.
- [ ] Specific edits or patch plan are included.
- [ ] Missing repo access, command output, docs, or examples are marked pending.
