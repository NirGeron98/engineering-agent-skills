# Security Incident Scoping Checklist

- [ ] Trigger, exact timestamps, and source of suspicion are recorded.
- [ ] Known facts (evidence-backed) are separated from unknown/suspected facts.
- [ ] Evidence preservation checklist is produced before any destructive containment step.
- [ ] Affected assets/users/systems are marked confirmed vs suspected based on evidence.
- [ ] Containment options include safety framing, approval/ownership, and rollback/verification.
- [ ] Escalation path names who and when.
- [ ] Follow-up investigation plan specifies next evidence to gather.
- [ ] Missing logs, timelines, or affected-asset lists are marked pending.
- [ ] No offensive response or unauthorized-access instructions are present.
