# Sample Task

## User prompt

"Create onboarding docs for new developers joining this repo."

## Abbreviated run

- Inspect README, scripts, tests, CI, docs, architecture notes, recent PRs, and ownership files.
- Build a repo map and first-day setup/test path.
- Obstacle: deployment ownership is not documented.

## Evidence requests

- `npm install`
- `npm test`
- Output or link for local service startup.
- Owner/escalation details for deployment if not in repo.

## Expected output

An onboarding doc outline or draft with repo map, first-day path, verified commands, common workflows, troubleshooting, ownership gaps, and contribution steps.

## Why this passes the quality bar

The doc is based on repository evidence, keeps advanced detail linked, and marks unverified setup/ownership items pending.
