# Test Strategy Checklist

- [ ] Scope, users/consumers, critical paths, release constraints, and failure impact are defined.
- [ ] Existing tests, CI commands, contracts, fixtures, closest precedents, and runtime constraints were inspected or exact evidence requests were emitted.
- [ ] Risk matrix covers likelihood, impact, observability, regression history, and ownership.
- [ ] Recommended unit, integration, contract, e2e, regression, and manual checks are tied to specific risks.
- [ ] Tests not worth adding yet are named with rationale.
- [ ] CI cost, flake risk, data/fixture needs, and runtime limits are addressed.
- [ ] Every recommended test has a verification command or pending command request.
- [ ] Missing repo access, test output, command output, specs, or risk context is marked as pending.
