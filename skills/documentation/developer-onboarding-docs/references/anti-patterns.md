# Developer Onboarding Docs Anti-Patterns

## Onboarding encyclopedia

- **Symptom**: New developers are told to read every subsystem before acting.
- **Why it matters**: The doc slows first contribution.
- **Correction**: Provide a first-day path and link advanced details.

## Unverified setup path

- **Symptom**: Setup commands are copied from scattered docs without running or checking them.
- **Why it matters**: First-time users lose trust immediately.
- **Correction**: Verify commands or mark exact outputs pending.

## Invisible ownership

- **Symptom**: The doc explains code but not who reviews, owns, or supports it.
- **Why it matters**: New contributors get blocked.
- **Correction**: Add ownership, review, and escalation paths or flag gaps.

## Role-first overload

- **Symptom**: The doc assumes every newcomer needs every workflow.
- **Why it matters**: Users cannot choose a starting path.
- **Correction**: Organize by first task and common workflows.
