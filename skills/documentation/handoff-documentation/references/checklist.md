# Handoff Documentation Checklist

- [ ] Audience, objective, current status, and resume point are clear.
- [ ] Git status/diff, commits, tests, logs, docs, decisions, issue/PR context, or prior outputs were inspected or requested.
- [ ] Facts, decisions, assumptions, and pending questions are separated.
- [ ] Files or areas touched are listed when available.
- [ ] Verification includes commands/results, failed checks, skipped checks, or exact pending requests.
- [ ] Risks, blockers, unresolved questions, and next steps are explicit.
- [ ] The handoff is concise enough to resume from, not a raw transcript.
- [ ] Missing evidence is marked pending, not invented.
