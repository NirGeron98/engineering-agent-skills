# API Docs Review Checklist

- [ ] API scope, versions, consumers, and source of truth are identified.
- [ ] Docs/spec, routes/handlers, schemas/types, tests, auth middleware, SDKs, and generated artifacts were inspected or requested.
- [ ] Endpoint purpose, auth, request fields, responses, errors, pagination, filtering, and versioning are checked.
- [ ] Examples are verified against schemas, tests, implementation, or executable output where possible.
- [ ] Mismatches cite both documentation and implementation/spec evidence.
- [ ] Documentation defects are separated from API design concerns.
- [ ] Concrete example fixes are included.
- [ ] Missing runtime output, specs, generated docs, or implementation access is marked pending.
