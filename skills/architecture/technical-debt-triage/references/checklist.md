# Technical Debt Triage Checklist

Use this before the final handoff.

- [ ] Built a debt inventory from user list plus inspectable issues, TODOs, incidents, flaky tests, PR churn, code hotspots, or review pain.
- [ ] Collected evidence for impact, frequency, user/business impact, engineering cost, risk, blast radius, and cost of delay.
- [ ] Identified dependencies between debt items.
- [ ] Separated quick wins, strategic fixes, monitor items, defer items, and ignore items.
- [ ] Explained what not to fix yet and why.
- [ ] Suggested tracking issues with acceptance criteria or verification.
- [ ] Avoided turning prioritization into immediate refactor planning.
- [ ] If issue history, incident data, churn, ownership, or impact evidence were missing, emitted exact evidence requests and marked scores pending.
