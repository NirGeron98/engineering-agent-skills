# Architecture Review Checklist

Use this before the final handoff.

- [ ] Reviewed the proposal, current docs, diagrams, and at least representative code/interfaces where available.
- [ ] Listed evidence sources and separated facts from assumptions.
- [ ] Mapped components, responsibilities, boundaries, ownership, data flow, and runtime dependencies.
- [ ] Checked failure modes, scalability, operability, testability, deployability, and compatibility.
- [ ] Considered high-level security/privacy implications without claiming a full security audit.
- [ ] Compared at least one alternative or explained why no viable alternative was available.
- [ ] Tied tradeoffs to documented constraints, not personal preference.
- [ ] Named risks, open questions, and recommended changes with evidence.
- [ ] Handed specialist work to the right skill instead of absorbing it.
- [ ] If repo, diagrams, dependency graph, ADRs, or runtime data were missing, emitted exact evidence requests and marked dependent conclusions pending.
