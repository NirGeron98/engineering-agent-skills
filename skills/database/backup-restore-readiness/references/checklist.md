# Backup Restore Readiness Checklist

- [ ] Actual backup schedule, retention, and encryption/access controls were inspected.
- [ ] Point-in-time recovery support/configuration was checked against RPO needs.
- [ ] The documented restore procedure was checked against actual backup format/location.
- [ ] Restore test history (last verified successful restore, and when) was checked, not assumed.
- [ ] RPO/RTO capability is compared against real evidence, not the backup schedule's label alone.
- [ ] Cross-region/cross-account storage was checked if relevant to the recovery scenario.
- [ ] Gaps are listed with a risk rating, not just a pass/fail summary.
- [ ] A restore test plan is included as a concrete next step.
- [ ] Ownership and follow-up deadlines are named.
- [ ] Missing backup config, retention settings, or restore test history is marked pending.
